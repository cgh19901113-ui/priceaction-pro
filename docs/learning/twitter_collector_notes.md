## 🎓 学到的经验

### 技术层面
1. **Playwright 是可靠的浏览器自动化方案**
   - 安装简单（pip install playwright）
   - 支持持久化 Cookie
   - 无头模式可后台运行

2. **Twitter 的反爬机制**
   - 必须登录才能查看内容
   - 注册需要手机验证（防止自动化注册）
   - 滚动加载是动态的（需要等待）

3. **数据提取技巧**
   - 推文元素：`article[data-testid="tweet"]`
   - 内容元素：`[data-testid="tweetText"]`
   - 时间元素：`time` 标签的 datetime 属性
   - 推文 ID：从 href 中的 `/status/` 提取
   - **重要**: 提取前移除 URL，避免假阳性

4. **股票代码识别**
   - A 股：6 位数字 + .ss/.sz（需要明确后缀）
   - 港股：5 位数字 + .hk
   - 美股：纯字母代码
   - 中文股票名需要映射表
   - **排除**: 加密货币（$BTC, $ETH 等）

### 工程层面
1. **自动化程度与数据质量的权衡**
   - 100% 自动化（RSS）→ 数据量太少
   - 95% 自动化（浏览器 + 一次登录）→ 数据量充足
   - 结论：接受 2 分钟手动操作，换取 10000+ 条数据

2. **增量更新的重要性**
   - 使用 tweet_id 去重
   - 只保存新数据
   - 支持中断后继续

3. **错误处理**
   - 网络请求可能超时
   - 元素可能不存在
   - 需要 try-except 包裹每个推文提取

4. **数据清洗至关重要**
   - URL 中的数字会被误判为股票代码
   - 需要在提取前移除 URL
   - 只保留带明确后缀的代码（.ss/.sz/.hk）
