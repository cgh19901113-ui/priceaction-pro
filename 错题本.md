### 错误 #10: vercel.json 配置缺失导致前端未部署
**时间**: 2026-04-10 02:01  
**错误信息**:
```
首页显示 FastAPI Swagger 页面，不是自定义前端
```

**原因**:
- vercel.json 为空配置 `{}`
- Vercel 自动检测 FastAPI 但不知道前端位置
- 没有配置 routes 路由规则

**修复**:
```json
{
  "builds": [
    {"src": "api/index.py", "use": "@vercel/python"},
    {"src": "frontend/**", "use": "@vercel/static"}
  ],
  "routes": [
    {"src": "/api", "dest": "api/index.py"},
    {"src": "/api/(.*)", "dest": "api/index.py"},
    {"src": "/", "dest": "frontend/index.html"},
    {"src": "/(.*)", "dest": "frontend/$1"}
  ]
}
```

**教训**: 
1. ⚠️ **vercel.json 不能省略**，必须明确配置构建和路由
2. ⚠️ **前后端分离项目必须配置 routes**
3. ⚠️ **部署后立即检查首页，不要只测试 API**
