## 错误记录

### 错误 #1: Vercel Handler 签名错误
**时间**: 2026-04-09 23:15  
**错误信息**: 
```
Error: Could not find a top-level "app", "application", or "handler" in "api/index.py"
```

**原因**: 
- 使用了错误的 `handler(request)` 签名
- Vercel Python runtime 不支持这种签名

**修复**:
```python
# ❌ 错误
def handler(request):
    symbol = request.args.get('symbol', '')

# ✅ 正确
def handler(environ, start_response):
    query_string = environ.get('QUERY_STRING', '')
```

**教训**: Vercel Python 使用 WSGI 标准，必须用 `handler(environ, start_response)`

### 错误 #2: Headers 格式错误
**时间**: 2026-04-09 23:37  
**错误信息**:
```
Failed to run "uv lock": No solution found when resolving dependencies
```

**原因**:
- Headers 用了字典格式 `{}`
- WSGI 要求元组列表 `[]`

**修复**:
```python
# ❌ 错误
headers = {
    'Content-Type': 'application/json',
    'Access-Control-Allow-Origin': '*',
}

# ✅ 正确
headers = [
    ('Content-Type', 'application/json'),
    ('Access-Control-Allow-Origin', '*'),
]
```

**教训**: WSGI headers 必须是 `[(key, value), ...]` 格式

### 错误 #4: URL 解析错误
**时间**: 2026-04-09 23:19  
**错误信息**:
```
AttributeError: 'WSGIRequest' object has no attribute 'args'
```

**原因**:
- 尝试用 `request.args.get()` (Flask/FastAPI 风格)
- WSGI environ 没有 args 属性

**修复**:
```python
# ❌ 错误
symbol = request.args.get('symbol', '')

# ✅ 正确
from urllib.parse import parse_qs
query_string = environ.get('QUERY_STRING', '')
query_params = parse_qs(query_string)
symbol = query_params.get('symbol', [''])[0]
```

**教训**: WSGI 用 environ['QUERY_STRING']，不是 request.args

### 错误 #6: 缺少 app 导出
**时间**: 2026-04-09 23:22  
**错误信息**:
```
WARNING: Due to builds existing in your configuration file...
Error: Could not find "app" or "application"
```

**原因**:
- Vercel 要求模块顶层有 `app` 或 `application`
- 只定义了 `handler` 函数

**修复**:
```python
# ✅ 添加
app = None  # Vercel requires 'app' at module level

def handler(environ, start_response):
    # ...
```

**教训**: Vercel Python 需要 `app` 或 `application` 变量

### 错误 #7: 未清理缓存文件
**时间**: 2026-04-09 23:45  
**错误信息**:
```
warning: in the working copy of 'api/index.py', LF will be replaced by CRLF
```

**原因**:
- 没有清理 `__pycache__/` 和 `*.pyc`
- Git 跟踪了不必要的文件

**修复**:
```bash
# 清理缓存
find . -name "__pycache__" -type d -exec rm -rf {} +
find . -name "*.pyc" -delete

# 添加到 .gitignore
echo "__pycache__/" >> .gitignore
echo "*.pyc" >> .gitignore
```

**教训**: 提交前必须清理缓存文件
